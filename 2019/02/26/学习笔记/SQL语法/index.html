SQL语法

发表于 2019-02-26 | 分类于 学习笔记

– DDL（数据定义语言）
– DML（数据操作语言）
– DCL（数据控制语言）
– 注意：SQL中大小写不敏感（大写和小写是一样的）;每条SQL语句必须以分号结束;

一、 DDL - 主要提供数据库和表的创建安、删除和修改

1. 创建数据库：create database 数据库名;

create database school;
– 如果不存在school2就创建它
CREATE DATABASE IF NOT EXISTS school2;
– 创建数据库的时候设置字符集编码，使数据库支持中文
CREATE DATABASE IF NOT EXISTS school DEFAULT CHARSET utf8;
– 删除数据库：drop database 数据库名
drop database school2;
– 如果school存在就删除
DROP DATABASE IF EXISTS school2;

2. 使用/切换数据库：use 数据库名;

USE school;

3. 新建表 CREATE TABLE 表名(字段名 类型1, 字段2 类型2,……);

– 注意：a. 表名一般需要加前缀’t’或者’tb’;
– b. 字段用来确定表中要存储哪些数据，字段名随便命名但是不能是关键字
– c. 数据类型必须是MySQL支持的数据类型
– 常用数据类型：int,char(size)-定长字符串,varchar(size)-不定长字符串,text-字符串，bit-布尔，date-日期
– CREATE TABLE IF NOT EXISTS 表名(字段名 类型1 约束1, 字段2 类型2 约束2.……)
– 常用的约束： not null - 不能为空， default 设置默认值，unique - 值唯一， primary key - 主键约束
– 主键约束：主键指的是表中能够唯一标识一条记录的字段(通过主键的值能够找到表中唯一一行记录)
– 注意：一个字段可以添加多个约束，多个约束之间用空格隔开

CREATE TABLE IF NOT EXISTS t_student
(
stuid int NOT NULL auto_increment,
stuname varchar(20) NOT NULL,
gender bit DEFAULT 1,
birth date,
PRIMARY KEY(stuid)
);

4. 删除表： DROP TABLE IF EXISTS 表名;

– 清空表中的记录：TRUNCATE TABLE 表名;
DROP TABLE IF EXISTS t_student;
TRUNCATE TABLE t_student;

5. 修改表

– 5.1 添加字段：ALTER TABLE 表名 add COLUMN 字段名 字段类型 约束;
ALTER TABLE t_student ADD COLUMN score FLOAT(5,2) DEFAULT 0;
– 5.2 删除字段：ALTER TABLE 表名 DROP COLUMN 字段名;
ALTER TABLE t_student DROP COLUMN score;

二、DML(数据库操作语言) – 主要针对数据库中数据的增、删、改、查

1、增（添加数据/记录）

– 1.1 插入数据/记录：insert into 表名 values (值1,值2,值3,……) - 依次给指定表中的字段赋值
INSERT INTO t_student VALUES(101,’张三’,0,’2019-02-25’)
– 1.2 插入数据/记录：insert into 表名(字段名1,字段名2,……) values(值1，值2,……) 以指定的顺序给指定的数据
INSERT INTO t_student (stuname,birth) VALUES (‘李四’,’2919-02-22’)
– 值的问题：sql中是数字对应的值直接写，字符串需要使用引号引起来，bit类型的值只有0或1，事件可以用内容是满足时间格式字符串也可以是通过时间函数获取的值
– 时间函数：now() - 当前时间 date(now) - 当前日期 year(now) - 当前年 month(now())
INSERT INTO t_student (stuname,birth) VALUES (‘王五’,DATE(NOW()));
– 一次插入多条记录
INSERT INTO t_student (stuname,birth) VALUES (‘赵六’,DATE(NOW())) ,(‘秦七’,DATE(NOW()));

2. 删(删除数据/记录)

– delete from 表名; – 删除指定表中所有记录
– delete from 表名 where 条件语句; - 删除满足条件的记录
– SQL中的条件语句： = 判断是否相等，<>不等于，python中的!=一样，>=，<=
DELETE FROM t_student WHERE stuid=100;

3. 改（修改数据/记录）

– update 表名 set 字段=新值; 将指定表中所有行的指定列/字段赋值为新值
UPDATE t_student set birth=’1999-12-31’,gender=1;

– UPDATE 表名 SET 字段1=新值1, 字段2=新值2,…… WHERE 条件语句; - 将表中满足条件的行中指定字段的值赋为新值
UPDATE t_student set gender=0 WHERE stuname=’李四’;

– 通配符%:表示任意个数的任意字符(包括0个)
UPDATE t_student SET birth=’2000-01-01’ WHERE stuname LIKE ‘王%’;
– 通配符_：表示有一个任意字符
UPDATE t_student SET birth=’2333-01-01’ WHERE stuname LIKE ‘王_’; – 修改stuname只有两个字符，且第一个字符是‘王’对应的行的birth的值
– 注意：通配符只针对字符串有效

4. 查(获取数据)

– 4.1 直接查询
– select from 表名; - 获取指定表中所有的行和列
USE school ;SELECT FROM t_student;

– SELECT 字段名1,字段名2,…… from 表名; - 获取指定表中所有行指定的列
SELECT stuname, stuid FROM t_student;

– SELECT 字段名1, 字段名2,…… FROM 表名; - 获取指定表中所有行指定的列

– SELECT FROM 表名 WHERE 条件; - 获取指定表中所有满足条件的行所有列的数据
SELECT FROM t_student WHERE stuid>115;

– 4.2 列重命名
– SELECT 字段1 AS 新字段1， 字段2 AS 新字段2,…… from 表名;

USE school; SELECT stuid AS ‘学号’, gender AS ‘性别’ FROM t_student;

– 4.3 对查询结果重新赋值(一般针对布尔数据)
– SELECT IF (字段相关的条件语句，值1，值2) from 表名; - 查询指定字段，并且判断字段对应的值是0还是1，如果是1结果为值1，否则为值2
– 注意：这儿的if的用法是MySQL转悠的
USE school ;SELECT stuname,IF(gender,’男’,’女’) FROM t_student;
– 通用写法： CASE 字段 WHEN 值 THEN 新值1 ELSE 新值2 end
USE school; SELECT CASE gender
WHEN 1 THEN
‘男’
ELSE
‘女’
END AS ‘性别’ FROM t_student;

– 4.4 对列进行合并
– SELECT CONCAT( 字段1,字段2,…) FROM 表名;
SELECT CONCAT(stuname,stuid) AS ‘name_id’ FROM t_student;
SELECT CONCAT(stuname,’:’,gender) AS ‘name_id’ FROM t_student;
– 注意：数字和字符串数据合一合并，bit类型的数据不可以合并
– SELECT CONCAT(stuname,’:’,gender) AS ‘name_id’ FROM t_student;

– 4.5 模糊查询 - 查询的时候通过like条件来指定查询对象
SELECT FROM t_student WHERE stuname like ‘%李四%’;
– SQL中支持逻辑运算符and（逻辑与运算）和or（逻辑或运算）, NOT (逻辑非)
SELECT FROM t_student WHERE stuname LIKE ‘%飞%’ OR NOT stuid < 110;

– 4.6 排序(先按之前的任何语法进行查询在排序)
– SELECT FROM 表名 ORDER BY 字段; - 对查询结果按照指定字段的值进行升序排序
SELECT FROM t_student ORDER BY stuname;
– SELECT FROM 表名 ORDER BY 字段;
– 查询结果按照指定字段的值进行升序排序
– SELECT FROM 表名 ORDER BY 字段 ASC;
– 对查询结果按照指定字段的值进行升序排序
– SELECT * FROM 表名 ORDER BY 字段 DESC;
– 对查询结果按照指定字段的值进行降序排序

– 排序的时候通过在 ORDER BY 的后边加多个字段，进行联排。排序的时候前面的字段的优先级高些
– SELECT FROM 表名 ORDER BY 字段1 ASC, 字段2; 对查询结果先按照字段1的值进行升序排序，当两个一样的时候按照字段2的值进行降序排序
– 4.7 限制
– SELECT FROM 表名 LIMIT 数字; - 获取查询结果的前指定数量条记录
SELECT FROM t_student LIMIT 3; – 获取查询结果的前三条记录
– SELECT FROM 表名 LIMIT M OFFSET N; – 对查询结果跳过前N条数据，取M条数据出来
SELECT * FROM t_student LIMIT 3 OFFSET 4; – 跳过前4条数据，然后取3条数据
